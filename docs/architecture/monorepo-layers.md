# Monorepo Layers and Mapping

This document describes the high-level workspace shape for the rearchitecture. It is intentionally about boundaries and ownership, not implementation detail.

## Layer Model

### Workspace shell

The repository is organized around explicit top-level areas:

- `apps/` for executable applications
- `packages/` for reusable workspace packages
- `references/` for upstream governance
- `docs/` for architecture and ADRs
- `scripts/` for workspace-level helpers
- `tests/` for cross-package verification

### Runtime and operator layer

`apps/cli` is the user-facing entrypoint. It owns orchestration and command routing, but it does not own prompt content or platform-specific source definitions.

### Toolkit content layer

`packages/toolkit` owns the canonical asset model for skills, commands, and agents. It is the only source from which downstream artifacts should be generated.

### Platform adapter layer

`packages/platform-*` adapt toolkit content into platform-specific packaging, install, or generation outputs. They consume toolkit manifests and do not redefine content.

### Governance layer

`references/` tracks upstream sources, sync notes, and immutable snapshots. It documents provenance and review state, but it is not part of the runtime dependency graph.

## Old-to-New Mapping

| Legacy source model | New owner |
| --- | --- |
| `skills/` | `packages/toolkit/src/content/skills/` |
| `commands/` | `packages/toolkit/src/content/commands/` |
| `agents/` | `packages/toolkit/src/content/agents/` |
| `QWEN.md` | generated by `packages/platform-qwen` |
| root `AGENTS.md` | thin project context entry, not content source |
| `instructions.md` | generated artifact or platform packaging output |
| ad hoc upstream notes | `references/notes/` |
| ad hoc upstream captures | `references/snapshots/` |

## Practical Meaning

- Source content is edited once, in toolkit.
- Platform files are regenerated from manifests.
- Upstream provenance is tracked separately from product content.
- Documentation explains the workspace contract instead of duplicating it.
- Root `AGENTS.md` exists only as a thin context shell for coding agents.
