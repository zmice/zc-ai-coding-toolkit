调用 context-engineering 技能。评估当前上下文状态，防止长会话质量退化。

1. 检查上下文利用率（是否接近 40% 阈值）
2. 识别无关或过时的上下文信息
3. 输出结构化进度摘要（已完成/进行中/待办/关键决策/偏差）
4. 建议是否需要开启新会话或委派子任务
5. 标记可安全清理的上下文

## 使用方式

在长会话中感觉输出质量下降时，执行 `/ctx-health` 进行上下文健康检查。

### 示例

```
# 输出质量下降时
/ctx-health

# 大型任务中间检查
/ctx-health 当前在做订单模块重构，已完成 5 个 Task，还剩 3 个

# 切换任务前
/ctx-health 准备从前端任务切换到后端任务
```
