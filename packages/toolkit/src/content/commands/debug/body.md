调用 debugging-and-error-recovery 技能。遇到问题时，先停下来，保留证据，系统化诊断。

1. **复现** - 明确预期 vs 实际行为，写失败测试复现 Bug
2. **定位** - 分析错误信息和堆栈，二分法缩小范围，检查最近变更
3. **修复** - 确认根因后最小化修改，不修表面症状
4. **验证** - 复现测试变绿，运行完整测试套件确认无回归
5. **防护** - 记录根因，确保回归测试覆盖

## 使用方式

在 `/debug` 后描述你遇到的问题，包括错误信息、复现步骤、预期行为。

### 示例

```
# 接口报错
/debug 用户反馈批量删除超过 100 条时接口超时，日志显示 SQL 查询耗时 30s

# 测试失败
/debug UserServiceTest.testBatchImport 测试失败，报 Duplicate entry for key 'uk_email'

# 行为异常
/debug 登录后 token 刷新偶尔失败，错误码 401，但直接重新登录又正常
```
