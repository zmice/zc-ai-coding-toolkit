调用 verification-before-completion 技能。未运行验证命令，不得声明完成。

1. 明确验证目标 - 你要声明什么（测试通过？构建成功？Bug 已修复？）
2. 执行验证命令 - 新鲜执行，不依赖历史结果
3. 完整读取输出 - 检查退出码，对比期望状态
4. 带证据报告 - 通过则附证据，不通过则报告具体失败点
5. 需求对照（可选） - 逐项检查 Spec / Plan 的每一条是否满足

## 使用方式

在完成任何实现工作后使用。描述你要验证的内容，我将执行完整的验证流程。

### 示例

```
# 功能实现后验证
/verify 确认用户登录功能的所有测试通过，构建成功

# Bug 修复后验证
/verify 确认批量导入的重复数据问题已修复，回归测试全部通过

# Spec 对照验证
/verify 对照 spec.md 逐项检查文件上传功能的实现完整性
```
