调用 planning-and-task-breakdown 技能。只读模式分析，不写代码。

1. 读取 Spec 和相关代码，识别依赖图
2. 垂直切片 — 每个任务交付可工作、可测试的功能
3. 每个任务 ≤ 200 行变更，不超过 ~5 个文件
4. 标注依赖关系、所需上下文和风险点
5. 编写验收标准和验证步骤
6. 按依赖顺序排列，标注并行可能性
7. **门控：等待人类确认计划后再继续**

## 使用方式

在 `/task-plan` 后提供 Spec 或需求描述，我将拆解为可执行的原子任务列表。

### 示例

```
# 基于 spec 拆解
/task-plan 按照 spec.md 的要求拆解文件上传功能的任务

# 直接描述需求
/task-plan 拆解用户权限管理模块：RBAC 模型、角色分配、菜单权限、数据权限

# 大型重构
/task-plan 规划单体应用拆微服务的任务，优先拆分用户模块和订单模块
```
