## 概述
将需求或规格拆解为可执行的原子任务列表，每个任务明确、独立、可验证。

## 拆解原则
1. **原子性** — 每个任务 ≤ 200 行变更，单一职责
2. **依赖明确** — 标注前置任务和所需上下文文件
3. **可验证** — 每个任务有明确的完成标准
4. **上下文友好** — 每个任务块在 40% 上下文窗口以内

## 输出格式
```
Task 1: [任务名称]
  文件: [涉及的文件]
  依赖: [前置任务]
  描述: [具体要做什么]
  验证: [如何确认完成]

Task 2: ...
```

## 注意事项
- 按依赖顺序排列任务
- 标注风险点和可能的阻塞
- 如需求不清晰，先列出需要澄清的问题
