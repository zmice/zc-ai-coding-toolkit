---

调用 idea-refine 技能。通过结构化的发散和收敛思维，将粗糙想法变为可执行方案。

1. 理解想法 — 你想解决什么问题？目标用户是谁？成功是什么样子？
2. 约束分析 — 时间资源限制、技术栈约束、需要兼容的现有系统
3. 方案探索 — 列出 2-3 个可行方案，对比 Pros/Cons，推荐最佳方案
4. 输出 — 结构化的需求描述、推荐技术方案、下一步行动项

## 使用方式

在 `/idea` 后描述你的想法（无论多粗糙），我将通过提问帮你细化为可执行的方案。

### 示例

```
# 粗糙的想法
/idea 我想给管理后台加一个操作审计日志功能

# 技术方向
/idea 我们的单体应用想拆微服务，日活 10w

# 产品特性
/idea 想做一个智能推荐系统，根据用户行为推荐商品
```
