## 概述
启动完整的 SDD+TDD 开发流程。从需求分析到规格编写、任务拆解、TDD 增量构建、代码审查，全流程门控推进。

## 工作流程

### 阶段 1：Spec（规格）
1. 分析需求，列出假设和约束
2. 定义成功标准（可测试的验收条件）
3. 编写技术规格说明
4. **门控：等待人类确认 spec 后再继续**

### 阶段 2：Plan（计划）
1. 将 spec 拆解为原子任务（每个任务 ≤ 200 行变更）
2. 标注每个任务的依赖关系和所需上下文
3. 确定测试策略
4. **门控：等待人类确认计划后再继续**

### 阶段 3：Build（构建）
对每个任务执行 TDD 循环：
1. 写失败测试（Red）
2. 写最小实现使测试通过（Green）
3. 重构改善代码质量（Refactor）
4. 验证所有测试通过
5. **门控：每个任务完成后输出摘要**

### 阶段 4：Review（审查）
1. 五维度代码审查：正确性 / 可读性 / 架构 / 安全 / 性能
2. 检查偏差日志
3. 运行完整测试套件
4. **门控：无 Critical 问题才算完成**
