## 概述
规划和执行代码迁移或 API 废弃，确保平滑过渡，不破坏现有功能。

## 迁移流程

### 1. 影响评估
- 识别所有受影响的调用方
- 评估变更范围和风险
- 确定是否需要兼容期

### 2. 迁移计划
- 编写迁移指南
- 准备自动化迁移脚本（如可行）
- 设计兼容层/适配器（渐进迁移）

### 3. 执行
- 添加废弃标注（@deprecated + 迁移说明）
- 实现新 API，保留旧 API 兼容
- 逐步迁移调用方
- 设定旧 API 移除时间表

### 4. 验证
- 确认所有调用方已迁移
- 运行完整回归测试
- 清理废弃代码

## 使用方式
描述你要迁移或废弃的内容，我将制定详细的迁移方案。
