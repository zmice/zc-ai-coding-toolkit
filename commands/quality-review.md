## 概述
对代码变更进行五维度系统化审查，输出结构化的审查报告。

## 审查维度

### 1. 正确性
- 逻辑是否正确？边界条件是否处理？
- 错误处理是否完善？
- 并发/竞态条件？

### 2. 可读性
- 命名是否清晰？代码是否自解释？
- 复杂逻辑是否有注释？
- 函数长度和复杂度是否合理？

### 3. 架构
- 是否符合现有架构模式？
- 职责划分是否清晰？
- 是否引入不必要的耦合？

### 4. 安全性
- 输入验证？SQL 注入？XSS？
- 认证和授权是否正确？
- 敏感数据处理？

### 5. 性能
- 是否有明显的性能问题？
- N+1 查询？不必要的循环？
- 资源泄漏风险？

## 输出格式
```
🔴 Critical: [必须修复]
🟡 Warning:  [建议修复]
🟢 Note:     [可选改进]
```
