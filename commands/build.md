## 概述
按 TDD 红绿重构循环进行增量构建。每次只实现一个任务，确保系统始终可编译、测试通过。

## TDD 循环
对每个任务严格执行：

1. **Red** — 先写失败测试，定义期望行为
2. **Green** — 写最小实现使测试通过（不多不少）
3. **Refactor** — 改善代码质量，保持测试绿色

## 质量要求
- 每个增量后运行完整测试套件
- 新代码必须有对应测试
- Bug 修复必须先写复现测试
- 原子提交，描述性消息

## 输出要求
每个任务完成后输出摘要：
```
✅ Task N 完成
  实现: [做了什么]
  测试: [X 个新测试，全部通过]
  变更: [涉及的文件]
```
