## 概述
同时激活 Careful + Freeze 的组合防护，适用于操作生产环境。提供最高级别的安全保护。

## 对应技能
safety-guardrails

## 工作流程

### 1. 激活危险命令预警（Careful）
- 自动启用 `/careful` 的全部功能
- 所有危险命令执行前触发确认流程

### 2. 激活文件锁定（Freeze）
- 自动锁定用户指定的路径范围
- 如未指定路径，默认锁定常见敏感路径：
  - `*.env*`（环境变量）
  - `**/migrations/**`（数据库迁移）
  - `**/config/production*`（生产配置）
  - `docker-compose.prod*`（生产部署）

### 3. 额外防护
- 标记所有环境变量修改为高风险操作
- 数据库相关操作强制二次确认
- 部署相关命令要求显式确认目标环境

### 4. 防护状态
```
🛡️ Guard 模式已激活
  Careful: ✅ 危险命令预警已开启
  Freeze:  ✅ [N] 个文件已锁定
  Extra:   ✅ 环境变量修改标记为高风险
```

## 使用方式
- `/guard` — 激活组合防护（使用默认锁定范围）
- `/guard src/core/** *.env` — 激活并指定锁定范围
- `/unguard` — 解除所有防护
