## 概述
生成项目文档或架构决策记录（ADR），确保知识可传递。

## 文档类型
1. **ADR（架构决策记录）** — 记录重要的技术决策及其原因
2. **API 文档** — 接口说明、参数、示例
3. **README** — 项目概述、快速开始、开发指南
4. **代码注释** — 关键逻辑的行内说明

## ADR 模板
```markdown
# ADR-NNN: [决策标题]
- 状态: [提议/已接受/已废弃]
- 日期: [YYYY-MM-DD]
- 决策者: [谁]

## 背景
[什么问题需要决策？]

## 决策
[选择了什么方案？]

## 备选方案
[考虑了哪些其他方案？为什么没选？]

## 影响
[这个决策带来什么后果？]
```

## 使用方式
说明你需要哪种文档，我将按规范生成。
