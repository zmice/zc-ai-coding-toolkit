## 概述
触发 Sprint 回顾，统计产出数据、总结决策效果、识别瓶颈、输出改进清单。

## 对应技能
sprint-retrospective

## 回顾流程

### 1. 采集数据
- 统计 Git 数据：提交数、LOC 变化、测试覆盖率
- 汇总任务完成情况（完成/延期/取消）
- 收集构建和 CI 指标

### 2. What Went Well / What Didn't
- 识别本轮做得好的实践和决策
- 识别遇到的问题、阻塞和意外返工
- 标注重复出现的模式

### 3. Spec 偏差检查
- 对比最终实现与原始 Spec
- 记录偏差原因（需求变更 / 技术约束 / 遗漏）
- 评估偏差的合理性

### 4. 流程健康度评估
- TDD 遵循率（Red-Green-Refactor 执行情况）
- 验证通过率（一次通过 vs 返工次数）
- 代码审查覆盖率

### 5. 输出 Action Items
```
📊 Sprint 回顾摘要
  提交数: [N] | LOC: +[X] -[Y] | 测试覆盖: [Z%]
  ✅ Well: [做得好的事项]
  ❌ Pain: [需要改进的事项]
  🎯 Actions:
    1. [具体改进行动 + 负责人 + 截止时间]
    2. [具体改进行动 + 负责人 + 截止时间]
```

## 使用方式
在 Sprint 结束时执行 `/retro`，自动采集数据并引导回顾。
