调用 shipping-and-launch 技能。发布前逐项检查，定义回滚方案。

1. **代码质量** — 测试通过、审查完成、静态分析干净
2. **安全** — 无密钥泄露、依赖审计通过、认证就位
3. **性能** — Core Web Vitals 达标、无 N+1、图片和 bundle 优化
4. **基础设施** — 环境变量、迁移脚本、监控配置就绪
5. **文档** — README、CHANGELOG、ADR 已更新
6. 定义回滚方案后再部署

## 使用方式

在 `/ship` 后描述你的发布场景，我将根据项目情况逐项检查并输出发布就绪报告。

### 示例

```
# 发布前检查
/ship 准备发布 v2.1.0，包含支付模块重构和新的导出功能

# 灰度发布方案
/ship 设计灰度发布方案，新功能先对 5% 用户开放

# 回滚预案
/ship 制定发布回滚预案，确保出问题时能 5 分钟内回滚
```
