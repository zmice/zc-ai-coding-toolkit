---

调用 safety-guardrails 技能。进入 Careful 模式，对所有后续操作启用风险检测。

1. 持续监控即将执行的命令和代码变更
2. 识别高风险操作 — 文件删除、数据库 DROP、force push、环境变量修改等
3. 执行前显示风险评估（等级、影响范围、可逆性）
4. 高危操作需用户显式确认，中危操作显示警告
5. 对每个危险操作建议更安全的替代方式

## 使用方式

执行 `/careful` 激活预警模式，所有后续危险操作将触发确认流程。

### 示例

```
# 激活预警模式
/careful

# 操作生产环境前
/careful 接下来要执行数据库迁移脚本

# 批量操作前
/careful 准备批量修改 50 个配置文件
```
