## 概述
激活 Careful 模式，AI 在执行任何危险命令前显示风险警告并要求确认。

## 对应技能
safety-guardrails

## 工作流程

### 1. 激活危险命令识别
- 进入 Careful 模式，对所有后续操作启用风险检测
- 持续监控即将执行的命令和代码变更

### 2. 标记高风险操作
识别以下类型的危险操作：
- **文件系统:** `rm -rf`、批量删除、覆盖关键文件
- **数据库:** `DROP`、`TRUNCATE`、`DELETE` 无 WHERE 条件、schema 迁移
- **Git:** `force push`、`reset --hard`、`rebase` 已推送分支
- **环境:** 修改 `.env`、变更生产配置、修改 CI/CD 流水线
- **权限:** `chmod 777`、修改 sudo 配置

### 3. 执行前风险评估
```
⚠️ 危险操作检测
  命令: [即将执行的命令]
  风险等级: 🔴 高危 / 🟡 中危
  影响范围: [受影响的文件/数据/服务]
  不可逆性: [是否可回滚]
```

### 4. 要求用户确认
- 高危操作：必须用户显式输入确认
- 中危操作：显示警告并等待确认

### 5. 建议替代方案
- 对每个危险操作提供更安全的替代方式
- 例如：`rm -rf` → 先移到临时目录；`force push` → 先创建备份分支

## 使用方式
执行 `/careful` 激活预警模式，所有后续危险操作将触发确认流程。
